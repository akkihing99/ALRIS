# ALRIS

Cost-Efficient Active Learning for Referring Image Segmentation and Grounding.

> **Anonymous repository for double-blind review.** No author / institution
> information is included here; please do not deanonymize.

## Repository layout

```
ALRIS_release/
├── active_train.py              # main entry — DDP via torchrun
├── al/active_learning.py        # RandomSampling + RegionEntropy + cost utilities
├── annotation_tool/             # cost-table builders + pseudo↔GT matcher
├── demo/                        # screen-capture demo of the annotation UI
├── engine/engine.py             # training / validation / inference loops
├── model/                       # DETRIS + DINOv2 backbone
├── project/app_ours_annotation/ # Flask annotation UI (ours)
├── utils/                       # dataset, config, misc, tokenizer
└── config/refcoco_detris.yaml   # example configuration
```

## Setup

Python ≥ 3.8 is recommended (the LMDB loader relies on legacy
`pyarrow.serialize`, which only ships wheels for Python 3.8–3.11).

```bash
# (optional but recommended) fresh environment
conda create -n alris python=3.8 -y
conda activate alris

# core dependencies (training + optional annotation tool)
pip install -r requirements.txt

# External submodules (not standard PyPI). Only required if you
# regenerate pseudo / corrected masks, or launch the Flask annotation tool.
pip install git+https://github.com/facebookresearch/segment-anything.git
pip install groundingdino-py>=0.4
# ViP-LLaVA (used only by the annotation app and the text-cost script):
#   git clone https://github.com/mu-cai/ViP-LLaVA  # place beside this repo
```

The annotation web app also expects the `ViP-LLaVA` repository on
`PYTHONPATH`.

### Pretrained weights

Place these under `pretrain/`:
* `ViT-B-16.pt` — CLIP image+text backbone
* `dinov2_vitb14_reg4_pretrain.pth` — DINOv2 visual backbone

## Data

RefCOCO / RefCOCO+ / RefCOCO-g images + annotation splits + LMDBs (loader
format) are identical to ETRIS — follow that repo's data instructions for the
download / LMDB build pipeline:
[`kkakkkka/ETRIS`](https://github.com/kkakkkka/ETRIS).

After preparing data, the working directory should contain:

```
datasets/
├── images/train2014/                       # COCO train2014 images
├── lmdb/<dataset>/{train,val,testA,testB,test}.lmdb
├── lmdb_gsam_llava/<dataset>/train.lmdb    # pseudo-label LMDB (GSAM masks + LLaVA texts)
├── masks/<dataset>/                        # GT masks
├── correction/<dataset>/                   # corrected masks used for accepted samples
├── gsam_mask/<dataset>2/                   # pseudo masks (flat-numbered, V2)
├── sam_gt_mask/meta_<dataset>_train.json   # img → segment-id meta
└── meta/<dataset>_gsam_llava_meta.json     # img-name → list of pseudo indices
```

The training loop uses **two different mask sources**:

* `gsam_mask/<dataset>2/` is the Auxiliary mask pool — these are what each
  unlabeled sample looks like before annotation. Generated by GroundingDINO+SAM
  (see step 1 below).
* `correction/<dataset>/` is the mask the model trains on once a sample has
  been *accepted*. In practice this folder is filled by either (a) the
  outputs of our interactive annotation tool
  (`project/app_ours_annotation/annotated_masks_ours/`), or (b) an oracle
  proxy from `utils/make/make_sam_gt_mask.py`, which picks the SAM-automatic
  mask with the highest IoU to GT for each `seg_id` (the "best a SAM-assisted
  annotator could realistically reach"). See step 2 below.

`masks/<dataset>/` only holds raw GT masks

## One-time generation pipeline

### 1. Pseudo masks (GSAM)
Generate the pseudo-mask pool via GroundingDINO + SAM. The script writes both
the V1 layout (`gsam_mask/<dataset>/<img_base>/<k>.png`) and the V2 flat
layout (`gsam_mask/<dataset>2/<global_id>.png`) the training loop expects.

```bash
torchrun --nproc_per_node=4 utils/make/make_gsam_mask.py \
    --config config/refcoco_detris.yaml \
    --box_th 0.23 --text_th 0.23 --max_masks 7
# → datasets/gsam_mask/refcoco/<img>/<k>.png    (V1)
# → datasets/gsam_mask/refcoco2/<global_id>.png (V2, used by training)
# → datasets/meta/gsam_mask_meta_refcoco2.json
```

### 2. Corrected masks (oracle proxy)
For experiments that do not involve real human annotators, populate
`correction/<dataset>/` via the SAM-automatic best-IoU oracle:

```bash
python utils/make/make_sam_gt_mask.py \
    --config config/refcoco_detris.yaml \
    --out_dir datasets/sam_gt_mask/refcoco \
    --sam_ckpt pretrain/sam_vit_h_4b8939.pth
# → datasets/sam_gt_mask/refcoco/<seg_id>.png
```

Then point the config at this directory (set
`DATA.train_mask_root_cor: datasets/sam_gt_mask/refcoco`) or copy/symlink the
result into `datasets/correction/refcoco/`. To use *real* human annotations
instead, gather them with the Flask app
(`project/app_ours_annotation/`) and drop the resulting PNGs under
`datasets/correction/<dataset>/`.

### 3. Pseudo↔GT index map
Match each pseudo mask to a GT mask (IoU-greedy, IoU < 0.1 → `-1`):

```bash
python annotation_tool/make_pseudo_to_gt.py --dataset refcoco
# → datasets/meta/pseudo_to_gt_refcoco.json
```

### 4. Per-sample cost tables

```bash
# Mask cost (SAM-based correction simulation)
python annotation_tool/mask_annotation_sam.py --dataset refcoco
# → datasets/al_cost/mask3/refcoco/sam_correction_stats.json

# Text cost (ViP-LLaVA-based typing simulation, 4-GPU DDP)
torchrun --nproc_per_node=4 annotation_tool/text_annotation_info.py \
    --dataset refcoco \
    --mono_counts datasets/n_gram/english_mono.txt \
    --bi_counts   datasets/n_gram/english_bi.txt \
    --tri_counts  datasets/n_gram/english_tri.txt
# → datasets/al_cost/text3/refcoco/text_cost_{K}.json
```

The cost tables are only required when running with `cost_key in {ours, naive,
oursMask_typing, poly_oursText}`.

## Training

```bash
torchrun --nproc_per_node=4 active_train.py --config config/refcoco_detris.yaml
```

The script runs `len(target_pcts)` rounds. **Round 0** samples
`target_pcts[0]` × full-dataset cost worth of pseudo-labels uniformly at
random, trains for `epochs_list[0]` epochs, evaluates on val/testA/testB.
**Rounds 1+** apply the chosen active-learning sampler (`random` or
`region_entropy`) and repeat. No pre-trained 1%-checkpoint is needed — the
round-0 model becomes the round-1 selector.

Per-round outputs go to
`exp/<dataset>/alris/<exp_name>/round_<k>/`:

* `best_model.pth`, `last_model.pth`
* `labeled_set.json` — accepted GT indices so far
* `selected_score.json` — per-pseudo selection scores this round
* `summary.txt` — per-round mIoU / oIoU history across val / testA / testB

### Common config knobs

| Field (under `AL:`) | Meaning |
|---|---|
| `target_pcts`  | cumulative cost fractions per round (round-0 uses `[0]`) |
| `epochs_list`  | training epochs per round |
| `label_mode`   | `random` or `region_entropy` |
| `cost_key`     | which cost mode to load (see `al/active_learning.py`) |
| `reject_cost`  | charge applied when a pseudo has no GT match |

### Reproducing the paper

Use the provided `config/refcoco_detris.yaml` with
`target_pcts: [0.01, 0.02, 0.05, 0.1, 0.2]` and `label_mode: region_entropy`.
For RefCOCO+ / RefCOCO-g copy this yaml, swap `dataset`, the LMDB / mask
paths, and the `test_*` block.
